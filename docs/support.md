# SF Boost Support

Last updated: March 7, 2026

## Support channels

- Chrome Web Store contact email: use the verified publisher support email shown in the Store listing for installation, privacy, or account-specific support.
- GitHub Issues for bugs, feature requests, and reproducible product issues: https://github.com/nocebov/sf-boost-chrome/issues
- Project homepage and documentation: https://github.com/nocebov/sf-boost-chrome

## Initial response targets

- 1 business day for install blockers, auth/cookie failures, and privacy or security reports.
- 3 business days for general bug triage, questions, and feature requests.

## What to include in a support request

- Salesforce page URL or page type, sanitized if needed.
- Enabled module name.
- Steps to reproduce.
- Expected result and actual result.
- Browser version and SF Boost version.

## Enterprise rollout note

For enterprise pilots, include the admin packet with the rollout request so reviewers can quickly assess permissions, domains, storage usage, and the absence of any developer-operated backend.
